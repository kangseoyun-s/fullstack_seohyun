### 📌 프로젝트 개요
Java의 제어문(`if`, `switch`, `for`)을 활용하여 사용자 인증, 입출금, 계좌 삭제, 통계 조회, 미니게임 기능을 포함한 콘솔 기반 은행 시스템을 구현했습니다. 탐정 콘셉트를 도입하여 사용자 경험을 향상시키고, 재미 요소를 추가했습니다.

### 🔧 주요 기능
- **사용자 등록 및 인증**: 아이디와 비밀번호를 기반으로 사용자 인증 처리
- **입금/출금 기능**: 잔액 업데이트 및 입출금 통계 기록
- **등급 시스템**: 잔액에 따라 사용자 등급(명탐정, 수석 조사관, 견습 탐정) 부여
- **미니게임 이벤트**: 입출금 후 확률적으로 숫자 맞추기 게임 발생 → 성공 시 5% 보너스 지급
- **통계 조회**: 입출금 횟수, 총액, 현재 잔액 등 시각적 통계 제공
- **계좌 삭제**: 사용자 요청에 따라 계좌 정보 초기화

### 🧠 사용 기술 및 문법
- `for(;;)` 무한 루프를 통한 지속적인 메뉴 제공
- `if-else`, `switch-case` 문을 활용한 메뉴 분기 처리
- `Scanner` 클래스를 통한 사용자 입력 처리
- `Math.random()`을 활용한 이벤트 확률 제어
- `String.equals()`를 통한 문자열 비교 및 인증

### 🎯 학습 포인트
- 제어문을 활용한 프로그램 흐름 제어 능력 향상
- 사용자 입력 기반 로직 설계 및 예외 처리 경험
- 콘솔 UI 구성 및 사용자 친화적 메시지 설계
- 게임 요소를 접목한 기능 구현으로 창의적 사고 확장

단순한 은행 시스템을 구현하는 데 그치지 않고, 탐정 콘셉트와 미니게임을 접목함으로써 사용자와의 상호작용을 강화할 수 있었습니다. 제어문을 활용한 다양한 기능 구현을 통해 Java의 흐름 제어 능력을 실전에서 익힐 수 있었고, 콘솔 기반 프로젝트의 재미와 가능성을 체감했습니다.

### 💥 자주 발생하는 Java 오류

| 오류 메시지 | 원인 | 해결 방법 |
|------------|------|-----------|
| `cannot find symbol` | 변수나 메서드 이름 오타, 선언 누락 | 선언 여부 확인, 대소문자 주의 |
| `';' expected` | 세미콜론 누락 | 문장 끝에 `;` 추가 |
| `NoSuchMethodError: main` | `main()` 메서드 선언 오류 | `public static void main(String[] args)` 정확히 작성 |
| `NoClassDefFoundError` | 클래스 파일 누락 또는 클래스패스 오류 | `.class` 파일 존재 여부 및 경로 확인 |
| `illegal start of expression` | 문법 오류 (예: 잘못된 키워드 사용) | 문장 구조 점검, 괄호 확인 |
| `class, interface, or enum expected` | 괄호 불일치 또는 클래스 외부 코드 존재 | 괄호 개수 확인, 클래스 내부에 코드 작성 |

### 🔍 오류별 해결 방안과 예시

#### 1. `cannot find symbol`
- **원인**: 변수나 메서드 이름 오타, 선언 누락
- **예시**:
  ```java
  System.out.println(nmae); // 오타: name → nmae
  ```
- **해결**: 변수 선언 확인, 철자 수정
  ```java
  String name = "Java";
  System.out.println(name);
  ```

---

#### 2. `';' expected`
- **원인**: 세미콜론 누락
- **예시**:
  ```java
  int a = 10  // 세미콜론 없음
  ```
- **해결**: 문장 끝에 `;` 추가
  ```java
  int a = 10;
  ```

---

#### 3. `NoSuchMethodError: main`
- **원인**: `main()` 메서드 선언 오류
- **예시**:
  ```java
  static void main(String[] args) { } // public 누락
  ```
- **해결**: 정확한 시그니처 사용
  ```java
  public static void main(String[] args) { }
  ```

---

#### 4. `NoClassDefFoundError`
- **원인**: 클래스 파일 누락 또는 클래스패스 오류
- **해결**:
  - `.class` 파일이 존재하는지 확인
  - 컴파일: `javac MyClass.java`
  - 실행: `java MyClass`
  - 클래스 경로 설정: `-cp` 옵션 사용

---

#### 5. `illegal start of expression`
- **원인**: 문법 오류 (예: 잘못된 키워드 사용)
- **예시**:
  ```java
  public class Test {
      public void method() {
          int if = 5; // 예약어 사용
      }
  }
  ```
- **해결**: 예약어 사용 금지, 문법 구조 점검
  ```java
  int value = 5;
  ```

---

#### 6. `class, interface, or enum expected`
- **원인**: 클래스 외부에 코드 존재, 괄호 불일치
- **예시**:
  ```java
  int a = 10; // 클래스 밖에 존재
  public class Test { }
  ```
- **해결**: 모든 실행 코드는 클래스 내부에 위치
  ```java
  public class Test {
      public static void main(String[] args) {
          int a = 10;
      }
  }
  ```

---

### 🧠 팁: 오류를 줄이는 습관

- IDE의 자동완성 기능 적극 활용
- 컴파일러 경고 무시하지 않기
- 테스트 코드 작성으로 예외 상황 미리 점검
- 로그와 디버깅 툴로 흐름 추적
