6) SOLID 원칙
S: 단일 책임 - 한 클래스는 한 가지 일만 해요 (쿠기 굽기)
O: 개방 폐쇄 - 새로운 쿠키는 쉽게 추가 (확장은 가능하지만 수정 닫힘)
L: 리스코프 치환 - 모든 쿠키는 같은 방식으로 만들 수 있음
I: 인터페이스 분리 - 필요한 기능만 딱! 인터페이스 나누기
D: 의존 역전 - 구체적인 쿠키 말고 추상적인 쿠키에 의존
